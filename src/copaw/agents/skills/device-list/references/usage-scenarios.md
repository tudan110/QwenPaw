# 使用场景

本文档用于帮助 Agent 识别用户问题属于哪一类设备查询，并决定需要获取多少数据、如何组织回复。

## 使用方法

先判断用户意图属于哪一类，再决定：

1. 是只查总数、列表，还是要全量统计
2. 是否需要先获取 `total`
3. 回复用表格、文字摘要，还是图表

## 1. 基础查询

适用问法：

- “获取设备列表”
- “一共有多少台设备”
- “显示前 10 台设备”
- “看一下设备信息”

建议处理：

- 只看总数：先请求 `page_size=1`，读取 `total`
- 只看部分列表：直接查第一页
- 不要把整段原始 JSON 原样回给用户

建议输出：

- 1 句摘要
- Markdown 表格（名称、IP、状态、资源状态、型号、厂商、类型）

## 2. 型号相关

适用问法：

- “有哪些设备型号”
- “统计各设备型号的数量”
- “按型号分组展示设备”
- “查看华为/中兴的设备型号分布”

建议处理：

- 通常需要全量数据
- 先按 `modelId` 聚合，再补设备示例
- 如果用户同时限定厂商，先按厂商过滤，再按型号统计

建议输出：

- Top 型号摘要
- 型号统计表
- 必要时补柱状图

## 3. 设备状态相关

适用问法：

- “设备状态统计（在线/离线）”
- “有多少台设备在线”
- “列出所有离线的设备”
- “查看设备状态分布情况”

建议处理：

- 统计分布时，需要全量数据
- 列离线设备时，过滤 `delStatus=offline`
- 如果只问在线数量，只返回数量和占比即可

建议输出：

- 数量 + 占比
- 离线设备表格（如用户要求明细）
- 饼图适合做在线/离线分布

## 4. 资源状态相关

适用问法：

- “资源状态正常的设备有哪些”
- “找出资源状态异常的设备”
- “统计资源状态分布”

建议处理：

- 过滤字段：`resStatus`
- 对异常设备，优先列出名称、IP、厂商、型号、最后访问时间
- 如果异常数量为 0，要明确说明“当前没有异常设备”

建议输出：

- 异常数量摘要
- 异常设备表格
- 状态分布图

## 5. 设备类型相关

适用问法：

- “有哪些设备类型”
- “查看路由器有哪些”
- “按设备类型统计”

建议处理：

- 统计时按 `devClass` 聚合
- 查询单一类型时直接过滤
- 用户若问“交换机/路由器有哪些”，优先给列表而不是只有计数

## 6. 厂商相关

适用问法：

- “有哪些厂商的设备”
- “华为的设备有哪些”
- “按厂商统计设备数量”

建议处理：

- 统计时按 `vendorId` 聚合
- 查询单一厂商时先过滤再展示明细
- 回复里尽量把代码值转成可读名称，如 `HW -> 华为`

## 7. 特定设备查询

适用问法：

- “查看设备 xxx 的详细信息”
- “搜索设备名称包含 xxx 的设备”
- “IP 为 xxx 的设备信息是什么”

建议处理：

- 精确 IP：优先按 `manageIp` 精确匹配
- 名称关键字：按 `devName` 模糊匹配
- 结果很多时先给前 20 条，并说明总匹配数
- 单条精确命中时，展示较完整字段

## 8. 综合分析

适用问法：

- “帮我分析一下设备情况”
- “生成设备统计报告”
- “设备整体情况怎么样”

建议处理：

- 获取全量数据
- 最少覆盖：总数、在线/离线、资源状态、厂商分布、型号 Top、类型分布
- 若无明显异常，也要说明“未发现明显异常点”

建议输出结构：

1. 概览
2. 状态分析
3. 厂商分布
4. 型号 Top
5. 类型分布
6. 异常设备或观察结论

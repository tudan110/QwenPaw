# 数据分析指南

本文档给出常见设备数据分析套路，重点说明“先取哪些数据、怎么聚合、如何输出结论”。

## 通用流程

1. 明确用户要的是列表、筛选、统计，还是综合分析
2. 判断是否需要全量数据
3. 能用 `analyze_devices.py` 时，优先用现成汇总
4. 只有在现成 mode 不够时，再基于 `rows` 做过滤、分组、计数、排序
4. 先写结论，再给表格或图表

## 优先使用的 mode

- `summary`：综合概览
- `status`：设备状态分布
- `resource-status`：资源状态分布
- `vendor`：厂商分布
- `model`：型号分布
- `type`：类型分布
- `abnormal`：异常设备清单
- `search`：关键字或组合条件搜索

## 场景 1：获取设备列表

适用：用户只想看设备列表或前几条记录。

处理方式：

1. 查询第一页或指定数量
2. 提取关键字段
3. 按表格输出

建议结论模板：

- “当前返回 X 台设备记录，以下为前 X 条。”

## 场景 2：统计设备型号

处理方式：

1. 获取全量设备列表
2. 按 `modelId` 分组计数
3. 按数量降序排序
4. 补充每个型号的设备示例

建议输出列：

- 型号
- 数量
- 占比
- 厂商
- 设备示例

建议结论模板：

- “设备型号较集中，Top 3 型号合计占比 XX%。”

## 场景 3：统计设备状态

处理方式：

1. 获取全量设备列表
2. 按 `delStatus` 聚合
3. 计算在线率、离线率
4. 若用户要求明细，再列离线设备

建议结论模板：

- “当前共 X 台设备，其中在线 X 台、离线 X 台，在线率为 X%。”

## 场景 4：查询资源状态异常设备

处理方式：

1. 获取全量设备列表
2. 过滤 `resStatus = abnormal`
3. 按设备名称或最后访问时间整理
4. 输出异常设备清单

建议结论模板：

- “当前检测到 X 台资源状态异常设备，建议优先关注以下设备。”

若异常数为 0：

- “当前未发现资源状态异常设备。”

## 场景 5：按厂商统计

处理方式：

1. 获取全量设备列表
2. 按 `vendorId` 聚合
3. 做代码值到中文名称映射
4. 计算占比并排序

建议结论模板：

- “设备主要集中在华为/中兴等厂商，其中 X 厂商占比最高。”

## 场景 6：按类型统计

处理方式：

1. 获取全量设备列表
2. 按 `devClass` 聚合
3. 统计不同类型占比

建议结论模板：

- “当前设备以路由器/交换机为主，前两类合计占比 XX%。”

## 场景 7：综合分析报告

建议至少覆盖以下维度：

1. 设备总数
2. 在线/离线分布
3. 资源状态分布
4. 厂商分布
5. 型号 Top
6. 类型分布
7. 异常观察

推荐输出结构：

### 1. 概览

- 总数
- 在线率
- 异常数

### 2. 状态分析

- 设备状态分布
- 资源状态分布

### 3. 结构分析

- 厂商分布
- 型号 Top
- 类型分布

### 4. 异常与建议

- 离线设备
- 资源异常设备
- 简短观察结论

## 排序与筛选建议

- 统计表默认按数量降序
- 列表默认按原接口顺序；若用户要求异常优先，可把异常放前面
- 示例设备最多列 3 个，避免表格过宽

## 输出格式建议

- **列表查询**：表格
- **统计查询**：结论 + 表格 / 图表
- **综合分析**：分级标题 + 图表 + 表格
- **单设备查询**：结构化字段列表
